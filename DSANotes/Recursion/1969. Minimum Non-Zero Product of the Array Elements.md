/*
========================================================
        LEETCODE 1969
        Minimum Non-Zero Product of the Array Elements
========================================================

========================================================
1) PROBLEM SUMMARY
========================================================

We are given:

nums contains all integers from:

[1 , 2^p - 1]

Example:

p = 3

nums = [1,2,3,4,5,6,7]


We can perform operations:

- pick any 2 numbers
- swap bits at same positions

Goal:

Find MINIMUM NON-ZERO PRODUCT possible.


========================================================
2) FIRST WRONG THINKING (VERY COMMON)
========================================================

Most beginners think:

- create array
- try operations
- swap bits
- compute product
- greedy simulation
- brute force

BUT this is WRONG direction.


WHY?

Because:

The operation count can be huge.

Problem is NOT asking:
"simulate operations"

Problem is asking:
"What FINAL state is achievable?"


========================================================
3) MOST IMPORTANT MINDSET SHIFT
========================================================

DON'T ask:

"How do I perform operations?"

ASK:

"What do operations mathematically ALLOW?"


This is the key transition into advanced DSA.


========================================================
4) KEY OBSERVATION (INVARIANT)
========================================================

Bit swapping DOES NOT create/destroy bits.

It only redistributes them.


So:

TOTAL COUNT OF SET BITS remains SAME.


This is called:

INVARIANT


========================================================
5) WHAT ARE WE MINIMIZING?
========================================================

We want:

a1 * a2 * a3 * ...

minimum possible.


========================================================
6) PRODUCT MINIMIZATION INTUITION
========================================================

For fixed sum:

Uneven numbers produce SMALLER product.


Example:

2 * 5 = 10

1 * 6 = 6


Again:

3 * 4 = 12

1 * 6 = 6


So:

To minimize product:
- make some numbers VERY SMALL
- make some numbers VERY LARGE


========================================================
7) SMALLEST NON-ZERO NUMBER
========================================================

We cannot use:

0

because product becomes zero.


So smallest allowed value:

1


Binary:

001


So ideally:
we want MANY 1s.


========================================================
8) LARGEST NUMBER
========================================================

Largest possible number:

111....111

which equals:

2^p -1


Example:

p = 3

111 = 7


========================================================
9) IMPORTANT PAIR OBSERVATION
========================================================

For p = 3:

Pairs:

1 + 6 = 7
2 + 5 = 7
3 + 4 = 7


Products:

1 * 6 = 6
2 * 5 = 10
3 * 4 = 12


Minimum occurs at MOST UNEVEN pair:

1 and 6


========================================================
10) CORE FORMULA INTUITION
========================================================

Maximum value:

2^p -1


Optimal pair:

1 and (2^p -2)

because:

1 + (2^p -2)
=
2^p -1


This minimizes pair product.


========================================================
11) WHY KEEP ONE MAXIMUM NUMBER?
========================================================

One number:

2^p -1

is kept untouched.


Breaking it apart increases product.


========================================================
12) HOW MANY PAIRS?
========================================================

Total numbers:

2^p -1


One maximum reserved.

Remaining:

2^p -2


These form pairs.


Number of pairs:

(2^p -2)/2


Simplify:

2^(p-1)-1


========================================================
13) FINAL FORMULA
========================================================

Final minimum product:

(2^p -1) * (2^p -2)^(2^(p-1)-1)


THIS is the entire problem.


========================================================
14) WHY MODULO NEEDED?
========================================================

Numbers become ENORMOUS.


Example:

p = 60

Exponent itself becomes:

2^59 -1


Impossible to store.


So problem asks:

Return answer modulo 1e9+7


========================================================
15) MODULO PROPERTY USED
========================================================

(a*b)%MOD
=
((a%MOD)*(b%MOD))%MOD


This lets us continuously shrink numbers safely.


========================================================
16) WHY BINARY EXPONENTIATION?
========================================================

We need:

a^b

where exponent is HUGE.


Normal power:
O(n)

Binary exponentiation:
O(log n)


========================================================
17) BINARY EXPONENTIATION IDEA
========================================================

Instead of:

2^16
=
2*2*2*2...


We do:

2^16
=
(2^2)^8
=
(4^2)^4
=
(16^2)^2


Exponent keeps reducing by HALF.


========================================================
18) EVEN POWER FORMULA
========================================================

x^n = (x*x)^(n/2)


Example:

2^8
=
(2*2)^4
=
4^4


========================================================
19) ODD POWER FORMULA
========================================================

x^n = x*(x*x)^((n-1)/2)


Example:

2^9
=
2*(2*2)^4
=
2*4^4


========================================================
20) IMPORTANT MODULO LESSONS
========================================================

WRONG:

x*x % MOD

because:
x*x may overflow BEFORE modulo.


SAFE:

((x%MOD)*(x%MOD))%MOD


========================================================
21) FINAL OPTIMIZED CODE
========================================================
*/

class Solution {

    static final long MOD = 1000000007;

    public int minNonZeroProduct(int p) {

        /*
        Maximum value:
        2^p -1
        */

        long max = (1L << p) - 1;

        /*
        Second maximum:
        2^p -2
        */

        long secondMax = max - 1;

        /*
        Exponent count:
        2^(p-1)-1
        */

        long power = (1L << (p - 1)) - 1;

        /*
        Final formula:

        (2^p -1) *
        (2^p -2)^(2^(p-1)-1)
        */

        long ans =
                (modPow(secondMax % MOD, power)
                        * (max % MOD))
                        % MOD;

        return (int) ans;
    }

    /*
    Binary Exponentiation

    Computes:
    base^exp % MOD

    in O(log n)
    */

    public long modPow(long base, long exp) {

        long result = 1;

        while (exp > 0) {

            /*
            If current exponent bit is 1
            */

            if ((exp & 1) == 1) {

                result =
                        (result * base) % MOD;
            }

            /*
            Square the base

            IMPORTANT:
            modulo immediately after multiplication
            */

            base =
                    (base * base) % MOD;

            /*
            Divide exponent by 2
            */

            exp >>= 1;
        }

        return result;
    }
}
