/*
====================================================================
===================== MAXIMUM SUM SUBARRAY OF SIZE K ================
====================================================================

Question:
Given an array and an integer k,
find the MAXIMUM sum of any contiguous subarray of size k.

--------------------------------------------------------------------
EXAMPLE
--------------------------------------------------------------------

arr = [2, 5, 1, 8, 2, 9, 1]
k = 3

Possible windows:

[2,5,1]  -> sum = 8
[5,1,8]  -> sum = 14
[1,8,2]  -> sum = 11
[8,2,9]  -> sum = 19
[2,9,1]  -> sum = 12

Maximum = 19


====================================================================
========================== CORE INTUITION ===========================
====================================================================

We need:
SUM of every window of size k.

Brute force would repeatedly calculate sum again and again.

But observe carefully:

When window shifts by 1:

1 element leaves
1 element enters

So instead of recalculating entire sum,
we UPDATE the previous sum.


====================================================================
========================= MAIN IDEA =================================
====================================================================

NEW WINDOW SUM =
OLD WINDOW SUM
+ incoming element
- outgoing element


====================================================================
====================== VISUAL UNDERSTANDING =========================
====================================================================

arr = [2, 5, 1, 8, 2, 9, 1]
k = 3


------------------------------------------------
FIRST WINDOW
------------------------------------------------

[2, 5, 1]

sum = 8


------------------------------------------------
SLIDE WINDOW BY 1
------------------------------------------------

Old window:
[2, 5, 1]

New window:
[5, 1, 8]

What changed ?

Removed:
2

Added:
8

So:

newSum = oldSum + 8 - 2
       = 8 + 8 - 2
       = 14


------------------------------------------------
NEXT WINDOW
------------------------------------------------

[1, 8, 2]

Remove:
5

Add:
2

newSum = 14 + 2 - 5
       = 11


This is the entire Sliding Window concept.


====================================================================
======================== WHY THIS IS POWERFUL =======================
====================================================================

Instead of recalculating k elements every time,
we only do:

1 addition
1 subtraction

for each slide.


====================================================================
========================= COMPLETE FLOW =============================
====================================================================

arr = [2, 5, 1, 8, 2, 9, 1]
k = 3


====================================================================
===================== STEP 1 : FIRST WINDOW =========================
====================================================================

Take first k elements.

------------------------------------------------
WINDOW:
------------------------------------------------

[2,5,1]

sum = 2 + 5 + 1
    = 8

max = 8


====================================================================
===================== STEP 2 : SLIDE WINDOW =========================
====================================================================

Start from i = k

i = 3

Current element:
8


------------------------------------------------
REMOVE OLD ELEMENT
------------------------------------------------

Outgoing element index:
i-k = 3-3 = 0

arr[0] = 2

------------------------------------------------
ADD NEW ELEMENT
------------------------------------------------

arr[3] = 8


------------------------------------------------
UPDATE SUM
------------------------------------------------

sum = 8 + 8 - 2
    = 14

max = 14


====================================================================
===================== NEXT WINDOW ==================================
====================================================================

i = 4

Add:
arr[4] = 2

Remove:
arr[1] = 5

sum = 14 + 2 - 5
    = 11

max = 14


====================================================================
===================== NEXT WINDOW ==================================
====================================================================

i = 5

Add:
9

Remove:
1

sum = 11 + 9 - 1
    = 19

max = 19


====================================================================
===================== NEXT WINDOW ==================================
====================================================================

i = 6

Add:
1

Remove:
8

sum = 19 + 1 - 8
    = 12

max = 19


FINAL ANSWER:
19


====================================================================
======================= DRY RUN TABLE ===============================
====================================================================

-------------------------------------------------------------
WINDOW              CALCULATION                 SUM     MAX
-------------------------------------------------------------
[2,5,1]             initial                     8       8
[5,1,8]             8 + 8 - 2                  14      14
[1,8,2]             14 + 2 - 5                 11      14
[8,2,9]             11 + 9 - 1                 19      19
[2,9,1]             19 + 1 - 8                 12      19
-------------------------------------------------------------


====================================================================
======================== WHY THIS WORKS =============================
====================================================================

Adjacent windows overlap heavily.

Example:

[2,5,1]
[5,1,8]

Common elements:
5,1

Only:
2 removed
8 added

So recalculating entire sum is wasteful.


====================================================================
======================== TIME COMPLEXITY ============================
====================================================================

First window calculation:
O(k)

Sliding remaining windows:
O(n-k)

Total:
O(n)


====================================================================
======================== SPACE COMPLEXITY ===========================
====================================================================

No extra data structure used.

O(1)


====================================================================
======================== BRUTE FORCE ================================
====================================================================

Brute Force:

For every window,
calculate sum from scratch.

Time:
O(n*k)

Very inefficient for large arrays.


====================================================================
======================== SLIDING WINDOW =============================
====================================================================

Sliding Window means:

Instead of rebuilding window again and again,
reuse previous computation.

This is one of the MOST IMPORTANT DSA patterns.


====================================================================
======================== IMPORTANT OBSERVATION ======================
====================================================================

Fixed size window problems usually involve:

1. Sum
2. Average
3. Maximum
4. Minimum
5. Count

and are commonly solved using:
SLIDING WINDOW.


====================================================================
========================= COMMON MISTAKES ===========================
====================================================================

1. Forgetting first window calculation

Need initial valid window before sliding.


------------------------------------------------

2. Wrong outgoing index

Correct:
arr[i-k]

NOT:
arr[i-1]


------------------------------------------------

3. Updating max before changing sum

Always update sum first.


------------------------------------------------

4. Using nested loops

That becomes brute force O(n*k).


====================================================================
======================== MEMORY TRICK ===============================
====================================================================

Think of a moving train window.

When train moves:

One person exits.
One person enters.

Instead of recounting everyone,
just adjust the count.


====================================================================
====================== FINAL UNDERSTANDING ==========================
====================================================================

1. Calculate first window sum.
2. Slide window one step at a time.
3. Add incoming element.
4. Remove outgoing element.
5. Track maximum sum.

This converts:
O(n*k) -> O(n)

using Sliding Window optimization.

====================================================================
*/
